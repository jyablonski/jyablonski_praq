# Checklist

- do self-join `and amazon_transactions_b.created_at - amazon_transactions.created_at <= 7` type queries when looking for second purhcase type questions
- can create simple cols for row counts like so; `sum(case when date_trunc('month', fb_comments_count.created_at) = '2019-12-01' then 1 else 0 end) as dec_comments,`
- `split_part(business_address, ' ', 1)` can be handy to pull specific values out of strings. note that it's 1-indexed, so to grab the first value you specify 1
- `string_to_array(lower(contents), ' ')` can be used to take a string, split it by some denominator like empty space, and turn it into an array of strings
- `unnest(string_to_array(lower(contents), ' '))` can be used to take an array of strings and turn it into 1 row for every value that was generated by the unnest
- lag or lead requires you to specify the column you want to grab the previous value of `lag(date_visited)`, and then you can do the normal over (partition by xyz)` syntax
- row_number will never have gaps in its ranks
- dense_rank will cluster the same values up into the same rank, and not have any gaps afterwards. it will go 1, 2 (2 ppl), 3 etc
- rank will cluster the same values up into the same rank, and then have gaps for the rest afterwards. it will go 1, 2 (2 ppl), 4 etc. no `rank = 3` will exist becuase 2 people were at rank 2.
